# Contributing to PTV Protocol

Thank you for your interest in contributing to the PTV Protocol! This document provides guidelines for contributing to the specification.

---

## 1. How Can I Contribute?

### Reporting Bugs
Search existing issues first. When reporting, include clear steps to reproduce and environment details (OS, Python, TPM version).

### Suggesting Enhancements
Use the feature request template. Explain the use case and describe the proposed solution.

---

## 2. Development Setup

```bash
# Clone repository
git clone https://github.com/anandkrshnn/ptv-protocol-spec.git
cd ptv-protocol-spec

# Install dependencies
pip install -r requirements-dev.txt
```

---

## 3. Pull Request Process

1. **Fork** the repository and create a branch.
2. **Implement** changes and add tests.
3. **Format** code with `black`.
4. **Submit** PR with a clear description and link to related issues.

---

## 4. Coding Standards

- **Python**: Follow PEP 8 + type hints.
- **Commit Messages**: Use conventional commits (e.g., `feat: Add BFT timeout`).
- **Tests**: Minimum 80% coverage required for all PRs.
